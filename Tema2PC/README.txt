Tema consta in implementarea unui server care primeste mesaje de la udp cu un
anumit topic si clientii tcp conectati sau nu la server ,care sunt abonati la 
acel topic vor primi mesajul transmis de clientii UDP.In rezolvarea temei am 
folosit 3 structuri una pentru formarea mesajului ce va fi trimis fiecarui client tcp,o structura in care se pastreaza topicurile la care s-a dat subscribe pana acuma, si in structura de topic avem o structura de clienti aceasta pastrand id,socket-ul prin care s-a conectat si variabila contor SF.
La rezolvarea ca schelet al temei am folosit rezolvarea laboratorului 08(TCP cu clienti multipli),in acest schelet schimband , adaugand detalii necesare.
Serverul deschizand 2 cai ,una pentru tcp si alta pt udp asteapta conexiuni valide,odata conectat un client tcp se verifica daca este un client nou sau unul care s-a deconectat mai devreme,fiind un client nou se adauga file descriptorul in multime,astfel data viitoare cand acesta trimite un mesaj va fi receptionat ca un mesaj de la un client tcp.De la clienti tcp putem primi 3 mesaje null care inseamna ca s-a deconectat ,subscribe ... si unsubscribe.La primirea subscribe ului sau unsubscribelui clientul este adaugat/sters din lista clientilor abonati la un anumit topic.In momentul in care se primeste un mesaj UDP se formeaza mesajul ce trebuie transmis,compus din: ip+port+mesajul primit de la udp; se parcurge lista de topicuri ,gasind topic se returneaza lista de clienti ,apoi parcurgand lista se trimite rand ,pe rand mesajul format mai devreme.Fiecare mesaj primit de la server,tcp ul il va afisa si transforma in functie de tipul acestuia.In server s-a implementat si comanda de exit la care se inchid toti file descriptorii deschisi anterior si socketii de udp si tcp.
